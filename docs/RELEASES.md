# Canvas Courseware Releases

本文档用于归档已经完成的版本、阶段性成果和对应交付内容。每完成一个版本或一个明确阶段，应及时补充本文件。

## Release 01 - MVP 基线完成

### 版本目标

- 完成 Canvas Courseware SDK MVP 基础闭环。
- 打通项目列表、正式工作台、编辑态、预览态、本地持久化与 JSON 导入导出。
- 建立 UI 层与事件层解耦、文档模型与渲染引擎解耦的基础架构。

### 已完成范围

- 完成通用 schema、事件协议、store 与 controller 基础实现。
- 完成 `packages/core`、`packages/fabric`、`packages/vue` 与 `apps/playground` 基础骨架。
- 完成编辑态与播放态适配器。
- 完成项目列表页与正式工作台页。
- 完成 `Text / Image / Rect` 三类基础元素创建与编辑。
- 完成多 slide 管理、属性面板、层级列表与层级调整。
- 完成 timeline 基础配置 UI。
- 完成编辑 / 预览模式切换。
- 完成本地项目持久化与 JSON 导入导出。

### 关键修正

- 修复编辑态仍按节点初始 `visible` 状态隐藏后续步骤对象的问题，确保编辑态默认全显。
- 补齐编辑态步骤归属可视化。
- 修复工作台自适应布局问题，解决非预期滚动条与内容溢出。
- 重构工作台信息架构、三栏布局、滚动交互与默认信息密度。

### 已知说明

- 当前版本完成的是 MVP 基线与工作台可用性收口，不代表下一版本效率能力已全部完成。
- `BUG-05`：`Text` 对象缩放问题未在本阶段完成，后续继续评估。

## Release 02 - VNext 第一批完成

### 版本目标

- 完成下一版本第一批“编辑效率基础”能力。
- 建立统一的 Arco Design 页面壳层与通用交互语言。
- 补齐高频编辑能力，降低重复操作成本。

### 已完成范围

- 完成基于 Arco Design Vue 的页面壳层与通用 UI 重构。
- 完成项目列表页、工作台页、编辑壳层、预览壳层 Arco 化改造。
- 完成全局色彩、布局 token 与工作台视觉层级统一。
- 完成撤销 / 重做能力，并完成相关复测。
- 完成快捷键体系。
- 完成节点复制 / 粘贴 / 重复。
- 完成框选、多选、批量移动、对齐与分布。
- 完成 slide 管理增强：复制、拖拽排序、删除确认、后插入与命名。
- 完成图层管理增强：重命名、锁定 / 解锁、隐藏 / 显示、选中定位。

### 关键修正

- 修复撤销 / 重做按钮在变更后仍保持禁用的问题。
- 修复多选批量操作历史记录拆分问题，确保批量移动 / 对齐 / 分布支持单次撤销 / 重做。
- 多轮 UI 评审后持续压缩工作台顶部、侧栏和时间轴视觉层级，统一交互语言与组件风格。

### 未纳入本阶段

- 本地图片导入
- 页面背景配置
- `BUG-05` 重新评估与修复

### 当前后续

- 下一步进入 VNext 第二批：编排与内容能力。

## Release 03 - VNext 第二批完成

### 版本目标

- 完成下一版本第二批“编排与内容能力”收口。
- 补齐时间轴高频编排、本地图片导入与页面背景配置闭环。
- 打通编辑态、预览态、缩略图与本地持久化之间的背景与素材链路。

### 已完成范围

- 完成时间轴步骤重命名、排序、复制、删除、从当前步骤开始预览与一键重置当前 slide 播放状态。

- 完成本地图片导入，支持直接选择本地图片文件插入当前 slide，并在项目本地持久化后恢复。
- 完成页面背景配置，支持按 slide 设置背景色、背景图和基础填充方式。
- 打通页面背景在编辑态、预览态与缩略图中的一致渲染表现。

### 关键修正

- 修复时间轴从当前步骤切入预览时的前序状态恢复问题，确保预览可从指定步骤连续播放。
- 补齐新建页面、后插入页面、复制页面与快速切页等边界场景下的背景初始化与渲染一致性。
- 完成 `VNext-08` 与 `VNext-09` 产品 review 复测，验证自动保存、刷新恢复、预览渲染与缩略图链路稳定可用。

### 未纳入本阶段

- `BUG-05`：`Text` 对象缩放问题仍未纳入本阶段交付。
- 图片插入入口统一、组件属性更换图片、图片一键设为背景、编辑区右键菜单等效率型需求，留待下一版本规划。

### 当前后续

- 下一步进入新一轮版本规划，聚焦“编辑链路效率与图片操作直达”。

## Release 04 - 编辑链路效率与图片操作直达完成

### 版本目标

- 完成高频编辑链路优化，减少项目创建、图片处理、画布操作与组件管理中的重复往返。
- 打通图片插入、替换、设为背景、右键快捷操作与缩略图同步等高频体验闭环。
- 收口右上角组件浮层与页面设置抽屉化改造，让工作台主画布占比继续提升。

### 已完成范围

- 完成首页首屏信息精简，并支持在新建项目流程中直接设置项目名称和页面尺寸。
- 完成图片入口统一与属性面板换图能力，移除“图片 / 导入图片”的重复表达。
- 完成图片一键设为页面背景入口，支持上传后直接写入当前 slide 背景图。
- 完成编辑区右键菜单，并支持文本节点双击直接进入画布内编辑态。
- 完成保存 / 切页前的当前 slide 自动截图，同步更新编辑态、预览态与项目列表封面。
- 完成组件管理面板重构，将组件列表改为右上角独立浮层，支持拖拽排序改层级与高频动作外化。
- 完成右侧“页面设置”由常驻面板调整为按需打开的独立抽屉入口。

### 关键修正

- 完成 `VNext-10` 至 `VNext-15` 的产品 review 与复测收口，验证创建、换图、设背景、右键菜单、缩略图同步与图层浮层链路稳定可用。
- 多轮 UI 评审后持续压缩图层浮层的视觉重量，统一入口 icon、拖拽手柄、显隐 / 锁定图标与页面设置入口的交互密度。
- 修复图层浮层相关 icon 未注册、操作热区偏小与信息层级过重等问题，提升可见性与可点性。

### 未纳入本阶段

- `BUG-05`：`Text` 对象缩放问题未在本阶段处理，留到下一版本单独收口。

### 当前后续

- 下一步进入 `BUG-05` 遗留问题修复版本；新的功能需求需重新进入 `docs/REQUIREMENTS.md` 后再规划。

## Release 05 - 页面设置闭环与编辑一致性收口完成

### 版本目标

- 完成时间轴引用一致性治理、对象级右键菜单语义收敛与页面设置逻辑优化。
- 修复近期影响编辑稳定性的画布缩放与侧栏定位问题，收拢当前版本执行面。
- 为下一版本“页面级配置闭环 + 文本高频编辑效率”腾出纯规划态空间。

### 已完成范围

- 完成 `VNext-22` 时间轴引用一致性治理，统一删除节点、批量删除与“设为背景”链路下的时间轴清理规则，并补齐一致性回归。
- 完成 `VNext-23` 对象级右键菜单语义优化，收敛空白区 / 单选 / 多选 / 图片对象四类菜单边界，并补齐图片对象“更换图片”快捷入口。
- 完成 `VNext-24` 页面设置整体逻辑优化，统一“当前页设置”入口、作用域提示、基础信息 / 背景配置分组，以及“设为背景”后的回跳与结果回显。
- 完成 `BUG-2026-04-10` 两项交互修复，解决点击元素后画布误缩放，以及浏览器缩放后侧栏收起按钮跑偏的问题。
- 完成根目录 `README.md` 的系统结构图、编辑态 / 播放态链路图与文件职责补充，并完成 `COMMAND_TYPES` 常量收口治理，降低后续接续成本。

### 关键修正

- 统一节点删除、图片设为背景与时间轴引用清理链路，避免编辑态时间轴面板、自动保存数据与预览态结果出现不一致。
- 明确对象级右键菜单的上下文边界，避免对象快捷操作与空白区创建能力混杂。
- 将页面设置收敛为单一入口和单一作用域表达，并把背景配置、结果回显与快捷链路回跳统一到同一抽屉上下文。
- 稳定编辑区缩放基准与三栏布局断点，降低高分屏、浏览器缩放与侧栏切换下的布局抖动风险。
