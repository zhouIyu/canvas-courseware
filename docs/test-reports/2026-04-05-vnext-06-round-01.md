## 测试步骤与结果

### 用例 1：图层列表基础加载与操作基线

1. 从项目列表创建空白项目。
2. 在编辑模式下连续新增 `18` 个文本对象、`1` 个矩形对象和 `1` 个图片对象。
3. 手动保存项目。
4. 切换到右侧“组件列表”面板。
5. 验证图层列表共展示 `20` 个对象。

结果：通过。

### 用例 2：图层重命名、预览显隐与锁定主链路

1. 在“组件列表”中对首个图层执行“命名”，修改名称为“标题文案”。
2. 对同一图层依次执行“预览隐藏”“锁定”。
3. 验证图层卡片出现“预览隐藏”“已锁定”状态标签。
4. 手动保存项目。
5. 读取本地存储中的当前项目记录，验证首个节点的 `name` 为 `标题文案`、`visible` 为 `false`、`locked` 为 `true`。

结果：通过。

### 用例 3：快速定位当前选中节点

1. 在“组件列表”中选中最后一个图层对象。
2. 将右侧面板滚动容器 `.editor-side-body` 的 `scrollTop` 重置为 `0`。
3. 点击“定位当前选中”。
4. 验证右侧面板滚动容器从顶部滚动到当前选中项附近，本轮记录到 `scrollTop` 从 `0` 变化为 `2610`。

结果：通过。

### 用例 4：解锁、保存与刷新恢复

1. 回到首个图层对象，执行“解锁”。
2. 验证“已锁定”状态标签消失。
3. 手动保存项目。
4. 刷新页面后重新进入“组件列表”。
5. 验证首个图层仍显示名称“标题文案”与“预览隐藏”标签，且不再显示“已锁定”标签。
6. 再次读取本地存储，验证首个节点的 `locked` 已恢复为 `false`。

结果：通过。

### 用例 5：预览模式切换烟测

1. 在刷新恢复后点击顶部“预览”模式按钮。
2. 验证工作台可切换到预览模式，页面无报错中断。

结果：通过。
