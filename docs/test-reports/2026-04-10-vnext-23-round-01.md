## 测试结果

- `pnpm test:vnext-23` 已覆盖空白区 / 单选 / 多选 / 图片对象四类菜单上下文；空白区仍保留 `插入文本 / 插入矩形 / 插入图片 / 粘贴`，对象级菜单不再混入插入类操作。

- 图片单选右键菜单已补齐 `更换图片` 与 `设为背景` 两个图片专属操作，并与通用对象操作分组展示。
- 从图片右键菜单执行“更换图片”后，图片节点 `id` 保持不变，默认节点名称与资源标识同步更新到新图片文件名，当前选中态仍保留在原图片节点上，结果与属性面板换图规则一致。
- 联动回归 `pnpm test:vnext-13`、`pnpm test:vnext-20`、`pnpm test:vnext-21` 后，文本右键菜单、双击文本编辑、图片属性换图、自定义命名保留、图片右键设为背景与填充方式写回链路继续通过。
- `pnpm typecheck` 与 `pnpm build` 均通过；自动化执行期间未捕获控制台报错或页面级异常。

## 结论

- `VNext-23` 的核心目标已经完成，编辑区右键菜单现在能按空白区、单选对象、多选对象和图片对象语义稳定分流，图片快捷换图能力也已经补齐。
