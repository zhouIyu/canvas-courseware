## 根因分析

- 预览器的播放器适配器原先只在组件首次 `onMounted` 时挂载一次
- 导入 JSON 后，预览区域内的 `canvas` 会随着 `activeSlide` 的短暂失效被 Vue 卸载并重新创建
- 旧的 Fabric 播放器仍绑定在已经失效的旧 DOM 上，而新的 `canvas` 没有重新挂载播放器
- 因此页面上方的播放状态、步骤列表和 slide 列表还能正常显示，但“当前画布”区域只剩一个原始 `canvas` 节点，导致画面空白且点击无法推进

## 修改说明

- 文件：`packages/vue/src/preview/useCoursewarePreview.ts`
- 移除了“仅在 `onMounted` 挂一次播放器”的做法
- 新增对 `previewCanvasRef` 的监听
- 当预览 `canvas` 首次出现，或因导入文档被 Vue 重建后重新出现时，自动重新执行 `mountAdapter`
- 对同一个 DOM 节点增加去重判断，避免无意义重复挂载

## 测试人员复测步骤

1. 进入项目列表页。
2. 打开示例项目“古诗讲解示例”。
3. 点击“导出 JSON”，得到一份标准 JSON 文件。
4. 返回项目列表，点击“新建项目”。
5. 点击“导入 JSON”，导入刚才导出的标准 JSON 文件。
6. 等待页面提示“已导入 ... 并保存到本地项目”。
7. 点击顶部“预览”切换到预览模式。
8. 观察当前画布是否正常显示封面标题、作者和卡片。
9. 点击预览区域，确认播放状态能从“已播放 0 / 2 步”推进到“已播放 1 / 2 步”。
10. 继续点击或等待后续步骤，确认导入后的项目可按时间轴正常播放。
