## 缺陷明细

### BUG-VNEXT01-01 撤销 / 重做按钮在变更后仍保持禁用

- 状态：复测通过
- 严重级别：阻塞
- 影响范围：`VNext-01` 撤销 / 重做核心能力；当前会直接导致该功能对用户不可用，阻塞产品 review 通过。
- 前置条件：本地服务已启动，可访问 `http://127.0.0.1:32173/projects`。
- 复现频率：2/2

#### 复现步骤

1. 进入项目列表页。
2. 点击“新建项目”，进入编辑工作台。
3. 观察顶部“历史”分组中的 `撤销 / 重做` 按钮，初始均为禁用。
4. 点击“文本”，向当前页面新增一个 `Text` 对象。
5. 等待自动保存恢复到 `已保存`。
6. 再观察顶部 `撤销 / 重做` 按钮状态。

#### 期望结果

1. 新增对象后，`撤销` 应立即变为可用。
2. 执行一次撤销后，`重做` 应变为可用。
3. 新增页面、属性修改、移动、缩放、时间轴编辑等后续历史操作也应遵循同样规律。

#### 实际结果

1. 新增对象后，文档节点数已经变为 `1`，但 `撤销` 仍然保持禁用。
2. `重做` 同样保持禁用。
3. 在新增页面场景下同样复现：页面数已变为 `2`，但 `撤销 / 重做` 依旧都不可用。

#### 补充观察

- 该问题目前已经足以阻塞继续验证“移动 / 缩放 / 图层 / slide / timeline 的撤销重做闭环”，因为最基础的历史入口就没有对用户开放。
- 从现象上看，更像是“历史状态没有正确反映到 UI”，而不是新增对象 / 新增页面命令本身失败。
