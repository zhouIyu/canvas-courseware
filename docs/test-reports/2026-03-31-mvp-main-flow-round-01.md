## 缺陷明细

### BUG-01 工作台保存失败，项目修改无法持久化

- 状态：复测通过
- 严重级别：阻塞
- 影响范围：项目标题修改、手动保存、自动保存、对象编辑后的本地持久化；直接影响 PRD 中“本地项目持久化保存”和“刷新后恢复”的验收要求。
- 前置条件：本地服务已启动，可访问 `http://127.0.0.1:32173/projects`。
- 复现频率：3/3

#### 复现步骤

1. 进入项目列表页。
2. 点击“新建项目”，进入工作台。
3. 在顶部“项目标题”输入框中输入 `测试回归项目-A1`。
4. 等待超过 800ms，触发自动保存。
5. 观察右上角保存状态。
6. 点击“项目列表”返回列表页，或直接刷新当前工作台页面。

#### 期望结果

1. 右上角保存状态显示“已保存”。
2. 项目标题修改被成功写入本地存储。
3. 返回项目列表后应显示最新标题。
4. 刷新页面后仍能恢复最新修改内容。

#### 实际结果

1. 右上角保存状态显示“保存失败”。
2. 返回项目列表后，项目标题仍显示旧值“未命名课件”。
3. 刷新页面后，刚才修改的标题未被保留。

#### 补充观察

- 同样的问题也会出现在点击顶部“保存”按钮时。
- 同样的问题也会出现在新增文本对象并等待自动保存时。
- 说明当前工作台保存链路并非仅标题输入场景失败，而是整体保存能力异常。

#### 证据

- 截图：`docs/test-reports/assets/2026-03-31-mvp-main-flow-round-01/bug-01-save-failure.png`

#### 排查线索

- 测试侧观察到项目保存链路集中在 `apps/playground/src/projects/project-repository.ts`。
- 建议开发优先检查深拷贝与保存前标准化逻辑，重点关注 [project-repository.ts](/Users/admin/MyDemo/canvas-courseware/apps/playground/src/projects/project-repository.ts#L8), [project-repository.ts](/Users/admin/MyDemo/canvas-courseware/apps/playground/src/projects/project-repository.ts#L54), [project-repository.ts](/Users/admin/MyDemo/canvas-courseware/apps/playground/src/projects/project-repository.ts#L134)。

#### 回测结果

- 回测轮次：Round 02
- 回测状态：复测通过
- 回测结论：
  - 修改标题后，右上角保存状态显示“已保存”
  - 手动保存和自动保存均可正常落盘
  - 返回项目列表后可看到最新标题
  - 重新打开与刷新页面后，标题和新增文本对象都能恢复
- 回测证据：`docs/test-reports/assets/2026-03-31-regression-round-02/bug-01-regression-pass.png`

### BUG-02 新增时间轴步骤后，右侧面板会自动跳回“页面设置”

- 状态：复测通过
- 严重级别：中
- 影响范围：时间轴配置连续编辑效率下降；会打断当前 timeline 主操作链路。
- 前置条件：本地服务已启动，可访问 `http://127.0.0.1:32173/projects`。
- 复现频率：2/2

#### 复现步骤

1. 进入项目列表页。
2. 点击“新建项目”，进入工作台。
3. 点击“文本”，保证当前页面至少存在 1 个对象。
4. 点击右侧标签“时间轴”，进入时间轴管理面板。
5. 点击“新增步骤”。
6. 观察右侧标签和下方面板内容。

#### 期望结果

1. 当前标签应继续停留在“时间轴”。
2. 新创建的步骤应立即展示在时间轴面板内，并可直接继续编辑步骤名称、触发方式和动作配置。

#### 实际结果

1. 步骤数量确实从 `0` 变为 `1`。
2. 但右侧激活标签会自动跳回“页面设置”。
3. 用户必须再次手动点击“时间轴”，才能继续编辑刚创建的步骤。

#### 补充观察

- 再次手动切回“时间轴”后，已创建的步骤仍然存在，说明问题主要出在交互状态回跳，而不是步骤创建失败。
- 该问题虽然不阻塞底层数据生成，但会明显打断用户配置时间轴的连续性。

#### 证据

- 截图：`docs/test-reports/assets/2026-03-31-mvp-main-flow-round-01/bug-02-timeline-tab-reset.png`

#### 排查线索

- 测试侧观察到右侧标签存在基于选中状态的自动切换逻辑。
- 建议开发优先检查 [CoursewareEditor.vue](/Users/admin/MyDemo/canvas-courseware/packages/vue/src/editor/CoursewareEditor.vue#L133) 一段与标签自动切换相关的监听逻辑，确认它是否在时间轴步骤新增后错误覆盖了用户当前标签状态。

#### 回测结果

- 回测轮次：Round 02
- 回测状态：复测通过
- 回测结论：
  - 点击“新增步骤”后，右侧标签保持在“时间轴”
  - 新创建步骤会停留在当前面板内可直接编辑
  - 修改步骤名称、切换触发方式、继续新增动作后，标签都没有自动跳回“页面设置”
- 回测证据：`docs/test-reports/assets/2026-03-31-regression-round-02/bug-02-regression-pass.png`
