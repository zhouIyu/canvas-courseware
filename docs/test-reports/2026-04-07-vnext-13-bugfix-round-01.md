# 2026-04-07 VNext-13 缺陷修复说明 Round 01

## 文档状态

- 本文档用途：供测试人员优先回归 `VNext-13` 中“文本节点双击画布内编辑后内容未写回属性面板与本地持久化”的阻塞缺陷

## 问题来源

- `docs/test-reports/2026-04-07-vnext-13-round-01.md` 记录的阻塞缺陷显示：文本节点双击进入画布内编辑后，画布可见文本会变化，但属性面板、文档状态与本地持久化仍保留旧文本。
- 缺陷触发路径为：新建文本节点 -> 双击进入画布内编辑 -> 修改文本 -> 点击编辑区空白区域退出编辑。
- 本轮修复严格限制在该缺陷，不扩展到右键菜单的其他菜单项、多选右键菜单或撤销 / 重做链路。

## 修复范围

- 仅修复文本节点双击画布内编辑后，退出编辑时没有立即写回标准文档的问题。
- 仅补齐“点击非 canvas 区域时主动结束内联编辑”的退出链路，不调整文本节点的创建、复制、删除或预览逻辑。
- 不修改 `VNext-13` 以外的需求范围，不扩展到 `VNext-14` 的缩略图同步任务。

## 根因分析

- Fabric 文本对象在双击进入内联编辑后，用户点击编辑区外层空白区域时，当前实现没有显式请求结束编辑态，导致这条路径下退出编辑的时机滞后于页面读取属性面板与本地存储的时机。
- 适配层原本把最终文本提交写在 `text:editing:exited` 事件中同步执行；当这一步与 Fabric 自身的退出流程叠加时，存在在退出事件内部就触发控制器回写和画布重同步的风险，容易打断 Fabric 的原生退出收尾。
- 结果是：用户在画布内看到的新文本并不总能在“点击空白退出”后第一时间写回标准文档，属性面板和本地持久化会短暂甚至持续保留旧值。

## 修改说明

### 1. 调整文本编辑退出后的标准命令提交时机

- 文件：`packages/fabric/src/editor-adapter.ts`
- 文本编辑退出时改为优先使用节点元数据中的 `slideId` 解析回写目标，避免依赖瞬时的当前页面状态。
- 最终文本提交改为在 `queueMicrotask` 中派发 `adapter.text.changed`，先让 Fabric 完成自身的 `editing:exited` 收尾，再进入标准命令层和画布同步链路，避免退出事件内部重渲染造成的时序冲突。

### 2. 为点击非 canvas 区域补齐显式退出内联编辑能力

- 文件：`packages/fabric/src/editor-adapter.ts`
- 新增 `exitActiveTextEditing()`，在适配层内部统一结束当前文本对象的内联编辑态，避免 UI 直接操作 Fabric 对象。
- 文件：`packages/vue/src/editor/useCoursewareEditor.ts`
- 暴露语义化方法 `requestInlineTextEditingExit()`，让 Vue 壳层只表达“请求结束内联编辑”，仍由适配层决定如何处理画布对象。
- 文件：`packages/vue/src/editor/CoursewareEditor.vue`
- 在全局 `pointerdown` 收口中，当用户点击目标不在 `canvas` 内时，主动请求结束当前文本编辑态，确保“点击编辑区空白 / 侧栏 / 工具区”都能及时触发文本提交。

## 测试人员复测步骤

1. 进入项目列表页，新建空白项目并进入编辑态。
2. 在编辑区空白区域右键，执行“插入文本”。
3. 双击该文本节点进入画布内编辑态，将文本改为 `画布内双击编辑成功`。
4. 点击编辑区中不在 `canvas` 内的空白区域，退出编辑态。
5. 立即检查右侧“文本属性”中的“内容”字段，确认是否已经同步为新文本。
6. 继续执行一次右键菜单“重复所选”与“删除所选”，确认副本与剩余节点都继承新文本。
7. 刷新页面并重新进入当前项目，确认文本内容仍为新文本。
8. 切换到预览模式，确认预览态展示的文本也保持为新文本。
9. 额外建议回归：双击编辑后通过右键菜单、侧栏切换或工具区点击退出编辑时，文本是否仍能稳定写回。

## 预期复测结果

- 文本节点在画布内编辑后，点击非 canvas 区域即可立即结束编辑并写回标准文档。
- 右侧属性面板中的“内容”字段会立即显示最新文本。
- 本地持久化、刷新恢复、复制 / 重复后的副本与预览态展示都保持最新文本。
- 修复后不影响 `VNext-13` 已通过的右键菜单插入、重复和删除链路。

## 已知限制与剩余风险

- 本轮重点覆盖的是“点击非 canvas 区域退出编辑”的主阻塞路径；建议测试阶段继续重点关注通过右键菜单、侧栏或其他外层控件退出编辑时的时序稳定性。
