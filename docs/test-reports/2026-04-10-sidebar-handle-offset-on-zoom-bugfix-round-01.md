## 根因分析

- 根因判断：编辑器和预览器在相对宽松的桌面宽度下就提前切换到了单列堆叠布局，但侧栏收起按钮仍沿用三栏布局时期的绝对定位。
- 在编辑器里，这会让左侧页面栏收起按钮继续按固定栏宽计算 `left`，从而漂到中间画布区域；单列时左侧页面栏根节点高度也没有被一并放开，页面列表会出现挤压甚至接近消失的现象。
- 在预览器里，左右侧栏 handle 同样会停留在内容区中部，表现成按钮脱离侧栏边界、悬在缩略页或画布内容上方。

## 修改说明

- 将编辑器三栏改为单列堆叠的断点从 `1240px` 下调到 `920px`，预览器从 `1100px` 下调到 `920px`，避免常见浏览器缩放后的桌面宽度过早触发单列布局。
- 为真正进入单列布局的场景补齐侧栏 handle 的兜底定位：左侧 handle 固定到布局左上安全位，右侧 handle 固定到布局右下安全位，避免继续漂入主内容区。
- 在编辑器单列布局下追加对 `SlideRailPanel` 子组件的 `:deep()` 覆写，确保页面栏本体和列表高度随内容展开，不再出现父容器切成 `auto` 后子组件仍锁死 `height: 100%` 的情况。
- 新增窄桌面缩放专项回归脚本：使用 `960x760` 视口打开示例项目，分别断言编辑器与预览器的左侧 handle 仍贴近侧栏 / 主区分界线，并产出截图与结构化快照。

## 给测试人员的复测步骤

1. 打开任一课件项目，进入编辑模式。
2. 使用浏览器缩放或缩小窗口宽度，模拟接近 `960px` 的桌面可视宽度。
3. 观察编辑器左侧页面栏的收起按钮是否仍贴边显示，且不会漂到中间画布上。
4. 切换到预览模式，重复相同缩放场景，确认预览页左右侧栏 handle 也不会压到缩略页或画布内容上。
5. 继续把窗口压窄到触发单列布局，确认按钮仍可见且页面栏列表不会异常塌陷。

## 预期复测结果

- 常见浏览器缩放后的桌面宽度下，编辑器与预览器仍保持稳定的桌面布局，不会过早切成异常单列。
- 左侧页面栏按钮始终贴近侧栏边界，不会漂到画布或缩略页内容区。
- 真正单列布局下，左右 handle 仍位于可点击的安全位置，页面栏内容不会消失或被压扁。

## 已知限制与剩余风险

- 本轮专项回归重点覆盖了 `960px` 的缩放桌面宽度，以及人工目测的更窄单列兜底场景。
