## 根因分析

### BUG-01 工作台保存失败

- 工作台保存时会把当前页面状态传入 `projectRepository.save`
- 保存链路中的深拷贝优先使用 `structuredClone`
- 当传入对象包含 Vue 响应式代理时，`structuredClone` 会直接抛错，导致保存流程进入失败分支，右上角状态显示“保存失败”

### BUG-02 时间轴标签回跳

- 编辑器右侧标签存在基于页面和选中态的自动切换逻辑
- 原监听写法会在 `snapshot` 发生任意更新时重新执行，即使当前页面和选中数量没有真实变化，也会把标签重置到“页面设置”
- 时间轴步骤新增会更新 `snapshot.document`，因此会误触发该重置逻辑

## 修改说明

### BUG-01 工作台保存失败

- 文件：`apps/playground/src/projects/project-repository.ts`
- 调整 `clonePlainData`
- 在 `structuredClone` 失败时回退到 JSON 深拷贝，兼容 Vue 响应式代理对象
- 保持原有仓库接口和保存时机不变，只修复深拷贝兼容性问题

### BUG-02 时间轴标签回跳

- 文件：`packages/vue/src/editor/CoursewareEditor.vue`
- 调整右侧标签自动切换监听逻辑
- 改为只在“当前页真实切换”或“选中数量真实变化”时才覆盖标签状态
- 时间轴步骤和动画这类与选中态无关的文档更新，不再强制把标签切回“页面设置”

## 测试人员复测步骤

### BUG-01 工作台保存失败

1. 进入项目列表页。
2. 点击“新建项目”，进入工作台。
3. 在顶部“项目标题”输入框中输入 `测试回归项目-A1`。
4. 等待超过 800ms，观察右上角保存状态。
5. 点击“项目列表”返回列表页。
6. 再次打开刚才项目，或直接刷新工作台页面。
7. 额外回归一次：新增一个文本对象，等待自动保存后再刷新页面。

### BUG-02 时间轴标签回跳

1. 进入项目列表页。
2. 点击“新建项目”，进入工作台。
3. 点击“文本”，保证当前页面至少存在 1 个对象。
4. 点击右侧标签“时间轴”。
5. 点击“新增步骤”。
6. 观察右侧激活标签与下方面板内容。
7. 额外回归一次：继续在“时间轴”面板内修改步骤名称、切换触发方式、添加动作，确认标签不会在这些操作后自动跳走。

## 预期复测结果

### BUG-01 工作台保存失败

- 右上角保存状态应显示“已保存”
- 返回项目列表后应显示最新标题
- 刷新页面后应恢复最新标题与新增对象
- 手动保存与自动保存都应可正常落盘

### BUG-02 时间轴标签回跳

- 点击“新增步骤”后，右侧标签应继续停留在“时间轴”
- 新创建的步骤应立即留在当前面板内可编辑
- 后续修改步骤和动作时，不应因为无关的文档更新而跳回“页面设置”
