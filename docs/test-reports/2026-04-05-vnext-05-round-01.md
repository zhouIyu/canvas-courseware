## 测试步骤与结果

### 用例 1：空白项目内验证 slide 管理主流程

1. 从项目列表创建空白项目。
2. 在第 1 页新增 1 个文本对象，作为复制内容基线。
3. 对第 1 页执行“后插入”，确认新页创建为第 2 页并位于原页之后。
4. 将第 2 页重命名为“章节页”。
5. 复制第 1 页，确认“第1页 副本”插入到原页之后，并保留 `1 个对象` 元数据。
6. 将“第1页 副本”拖拽到“章节页”之后，确认顺序更新正确。
7. 以“章节页”为目标执行“后插入”，确认新页“第4页”插入到中间位置，而不是追加到末尾。

结果：通过。

### 用例 2：删除确认与激活页回退

1. 对处于激活态的“第4页”点击“删除”。
2. 验证出现删除确认浮层。
3. 先点击“取消”，确认页面未被删除。
4. 再次点击“删除”并确认，验证“第4页”被移除。
5. 验证删除后激活页回退到后续可用页“第1页 副本”。

结果：通过。

### 用例 3：slide 管理相关撤销 / 重做回归

1. 在删除“第4页”后执行“撤销”。
2. 确认“第4页”恢复，且页面顺序恢复为 `第1页 -> 章节页 -> 第4页 -> 第1页 副本`。
3. 执行“重做”。
4. 确认“第4页”再次删除，顺序回到 `第1页 -> 章节页 -> 第1页 副本`。

结果：通过。

### 用例 4：保存与刷新恢复

1. 在最终状态下手动点击“保存”。
2. 刷新页面。
3. 确认页面顺序仍为 `第1页 -> 章节页 -> 第1页 副本`。
4. 确认“第1页 副本”仍保留 `1 个对象` 元数据。

结果：通过。
