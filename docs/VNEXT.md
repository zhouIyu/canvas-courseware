# Canvas Courseware VNext Plan

本文档用于记录下一版本的规划、范围边界和开发批次，不直接替代当前执行中的 `docs/TODO.md`。

## 版本定位

- 当前执行池为空，`VNext-32` 已完成开发、自测与产品 review；版本归档可在本轮规划确认后单独收口。
- 下一版本建议切换为 `VNext-33`，聚焦“编辑高频效率 + 连续操作稳定性”。
- 版本目标是在不引入新元素类型和新媒体类型的前提下，优先补齐文本编辑、拖拽保存、元素复用、键盘操作、页面管理与预览验证中的高频短板。
- 当前版本规划继续以 `docs/REQUIREMENTS.md` 中已收集需求为输入来源；除需求池中的 2 条事项外，本轮额外补充 5 个功能 / 优化项。
- 在用户确认正式推进开发前，本页暂不同步到 `docs/TODO.md`；后续一旦启动下一版本，再由 `docs/TODO.md` 承接实际执行与状态跟踪。

## 已确认范围

- [ ] VNext-33-A 文本编辑态样式修改不中断编辑
  说明：
  围绕 `REQ-2026-04-13-01`，解决文本对象在编辑态下修改颜色、字号、加粗、斜体等样式时，被自动保存打断并退出编辑状态的问题。
  来源：
  `docs/REQUIREMENTS.md`
  需求拆分：
  1. 文本对象处于编辑态时，颜色、字号、加粗、斜体等高频样式修改不应强制退出当前编辑会话。
  2. `TextTool`、右侧属性面板与自动保存状态提示保持一致，不出现一处修改成功、另一处状态回退的问题。
  3. 刷新恢复后，文本内容与样式结果保持正确，不因编辑态优化引入数据丢失。

- [ ] VNext-33-B 拖拽元素自动保存策略优化
  说明：
  围绕 `REQ-2026-04-14-01`，重构拖拽链路中的自动保存触发时机，降低拖拽过程中的画布与侧栏闪动。
  来源：
  `docs/REQUIREMENTS.md`
  需求拆分：
  1. 拖拽过程中不再频繁触发导致界面明显闪动的保存写回。
  2. 拖拽结束后，自动保存仍能在合理时间内完成，并保证刷新恢复结果正确。
  3. 保存状态提示与诊断日志需要适配新的触发策略，避免拖拽过程中频繁闪烁或产生噪音日志。

- [ ] VNext-33-C 元素复制粘贴与快速重复
  说明：
  为 `Text / Image / Rect` 补齐同页内的高频复用能力，减少重复创建与重新录入属性的成本。
  来源：
  本轮版本规划补充
  需求拆分：
  1. 支持对当前选中元素执行复制、粘贴与快速重复，首批覆盖单选场景。
  2. 粘贴或重复生成的新元素默认带有可感知位移偏移，避免与原元素完全重叠。
  3. 新元素需保留基础视觉属性与尺寸位置规则，避免重复后还要重新配置主要样式。

- [ ] VNext-33-D 键盘快捷键与方向键微调
  说明：
  为编辑态补齐键盘操作闭环，提升基础排版与对象调整效率。
  来源：
  本轮版本规划补充
  需求拆分：
  1. 支持 `Delete / Backspace` 删除选中对象，支持 `Esc` 退出高频临时态。
  2. 支持方向键微调对象位置，并区分常规步进与加速步进。
  3. 快捷键行为需与文本编辑态隔离，避免文本输入过程中误删对象或误触画布级操作。

- [ ] VNext-33-E 对齐参考线与吸附
  说明：
  为拖拽排版补齐可视化参考线和吸附反馈，降低对象对齐成本。
  来源：
  本轮版本规划补充
  需求拆分：
  1. 拖拽对象时，支持与画布边缘、画布中心和相邻对象产生基础对齐参考。
  2. 满足吸附条件时提供清晰的视觉反馈，帮助用户快速确认对齐结果。
  3. 吸附规则需保持可预测，避免对象在拖拽过程中出现难以理解的跳动。

- [ ] VNext-33-F 页面复制、排序与快速新建
  说明：
  面向多页课件场景补齐页面级高频操作，降低重复搭页和调整页序的成本。
  来源：
  本轮版本规划补充
  需求拆分：
  1. 支持复制当前页，继承页面尺寸、背景与元素内容。
  2. 支持调整页面顺序，保证缩略页、编辑态与预览态页序表达一致。
  3. 新建页面时优先延续当前工作流上下文，减少重复配置基础页面信息的操作成本。

- [ ] VNext-33-G 预览播放控制增强
  说明：
  为预览态补齐更完整的验证链路，降低动画与时间轴调试成本。
  来源：
  本轮版本规划补充
  需求拆分：
  1. 在预览态提供更明确的上一页 / 下一页 / 重播当前页等基础控制。
  2. 提供当前页播放进度或步骤状态提示，帮助用户快速确认时间轴推进位置。
  3. 保持预览态仍然严格消费通用 timeline，而不是引入额外的播放态私有数据结构。

## 约束说明

- 下一版本仍需遵守 `docs/PRD.md` 中定义的架构边界，尤其保持 UI 层与事件层解耦、文档模型与渲染引擎解耦。
- 文本编辑态、拖拽保存、复制粘贴、键盘快捷键与页面级操作都必须通过标准命令层与状态层落地，不得直接把 Fabric 事件逻辑耦合进 Vue 组件。
- 自动保存策略调整后，需同步保证本地持久化可靠性、状态提示一致性与诊断日志可读性。
- 预览控制增强只能补强现有播放控制体验，不应引入第二套独立的播放数据模型。
- 当前已纳入版本规划但尚未进入执行池的需求，仍可暂时保留在 `docs/REQUIREMENTS.md`；待正式同步到 `docs/TODO.md` 后，再从需求池回收对应条目。
