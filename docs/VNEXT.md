# Canvas Courseware VNext Plan

本文档用于记录下一版本的规划、范围边界和开发批次，不直接替代当前执行中的 `docs/TODO.md`。

## 版本定位

- 当前版本已完成到 `VNext-21` 的开发交付，新的需求池已形成下一轮版本输入。
- 下一版本聚焦“编辑逻辑一致性与配置链路优化”，优先处理已收集的 4 条需求。
- 本轮版本不扩展新元素类型，而是集中优化时间轴引用清理、右键菜单语义和页面设置逻辑。
- 下一版本继续遵守 UI 层与事件层解耦原则，不引入新的引擎耦合型实现。

## 核心功能范围

- [ ] VNext-22 时间轴引用一致性治理
  说明：
  基于“删除元素后时间轴残留事件”的需求输入，重新梳理元素删除与 timeline 的联动规则，统一清理对象点击触发引用、步骤动作目标和动画 `targetId`，覆盖删除元素、删除图层、批量删除和图片转背景等会移除节点的入口。
  需求拆分：
  1. 梳理时间轴中所有节点引用类型，包括触发对象、动作目标与动画目标。
  2. 统一删除入口下的时间轴清理规则，覆盖删除元素、删除图层、批量删除和图片转背景。
  3. 当步骤失去全部有效动作或触发闭环时，自动删除步骤，而不是保留空步骤。
  4. 补齐编辑态时间轴面板、存储数据与预览态播放的一致性回归。
  执行子任务：
  VNext-22-A 引用模型盘点：梳理 timeline 中所有节点引用位置，形成删除影响清单。
  VNext-22-B 删除入口统一：统一删除元素、删除图层、批量删除、图片转背景等入口下的时间轴清理规则。
  VNext-22-C 空步骤自动删除：当步骤失去全部有效动作或触发闭环时自动删除步骤，并保证时间轴顺序正确收敛。
  VNext-22-D 一致性回归：回归编辑态时间轴面板、存储数据和预览态播放结果的一致性。
- [ ] VNext-23 对象级右键菜单语义优化
  说明：
  基于右键菜单上下文分离需求，按空白区 / 单选对象 / 多选对象的上下文重整菜单项；当已选中元素时隐藏插入类操作，同时为图片对象补齐“更换图片”快捷入口。
  需求拆分：
  1. 明确空白区、单选对象、多选对象三类菜单的显示规则。
  2. 对象级菜单移除插入类操作，避免与空白区创建能力混杂。
  3. 为图片对象菜单补齐“更换图片”入口，并与“设为背景”等图片专属操作统一排序。
  4. 回归单选 / 多选 / 图片对象 / 空白区菜单的一致性与可预期性。
  执行子任务：
  VNext-23-A 菜单上下文矩阵：梳理空白区、单选对象、多选对象三类菜单的显示规则和操作边界。
  VNext-23-B 对象级菜单收敛：移除对象级菜单中的插入类操作，确保对象操作与创建操作彻底分流。
  VNext-23-C 图片专属快捷操作补齐：为图片对象菜单补齐“更换图片”入口，并与“设为背景”等图片专属操作统一排序。
  VNext-23-D 菜单一致性回归：回归空白区 / 单选 / 多选 / 图片对象菜单在显示、操作结果与状态回写上的一致性。
- [ ] VNext-24 页面设置整体逻辑优化
  说明：
  基于页面设置整体逻辑优化需求，重新梳理页面设置抽屉的入口、当前页上下文表达、页面名称 / 尺寸 / 背景配置的分组顺序，以及与“设为背景”等快捷入口之间的回跳和回显逻辑。
  需求拆分：
  1. 明确页面设置入口与当前页作用域提示。
  2. 重排页面名称、页面尺寸、背景配置等字段的分组和优先级。
  3. 统一页面设置与“设为背景”等快捷链路之间的回跳、回显和结果反馈。
  4. 清理重复入口、重复文案和低频项平铺问题，降低理解成本。
  执行子任务：
  VNext-24-A 页面设置入口与作用域提示：统一页面设置入口形态，并明确当前页上下文与作用范围提示。
  VNext-24-B 页面字段分组重排：重排页面名称、页面尺寸、背景配置等字段的分组、优先级和默认展开顺序。
  VNext-24-C 快捷链路回跳与回显统一：统一“设为背景”等快捷链路执行后的页面设置回跳、结果回显和状态反馈。
  VNext-24-D 页面设置信息减负：清理重复入口、重复文案和低频项平铺问题，降低页面设置理解成本。

## 暂不纳入下一版本

- 富文本
- 表格、图表、音视频
- 组合 / 打散
- 协同编辑
- 模板市场
- 图片 / PDF 导出
- 多引擎同时接入
- 新的功能型需求：待重新写入 `docs/REQUIREMENTS.md` 并完成评审后，再纳入后续版本规划
