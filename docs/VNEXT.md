# Canvas Courseware VNext Plan

本文档用于记录下一版本的规划、范围边界和开发批次，不直接替代当前执行中的 `docs/TODO.md`。

## 版本定位

- 下一版本聚焦“编辑链路效率与图片操作直达”，优先解决高频操作路径过长的问题。
- 版本目标是缩短项目创建、图片处理、画布快捷操作和组件编排的往返路径，让编辑动作更接近 PPT 式直达体验。
- 规划优先级以 `docs/REQUIREMENTS.md` 中已收集的需求为主，再结合依赖关系和交付闭环拆分实施批次。
- 下一版本继续遵守 UI 层与事件层解耦原则，不引入新的引擎耦合型交互实现。

## 核心功能范围

- [ ] 项目创建与首页入口优化
  说明：
  首页继续简化，降低首屏非必要信息占比；新增项目时，支持在创建流程中直接设置项目名称和页面尺寸。
- [ ] 图片插入与替换链路升级
  说明：
  统一“图片”入口命名，移除“图片 / 导入图片”的重复表达；选中图片组件后，支持在组件属性中直接更换图片资源。
- [ ] 图片一键设为页面背景
  说明：
  在上传图片流程或图片右键菜单中提供“设为背景”入口，让图片可直接应用到当前 slide 背景。
- [ ] 编辑区快捷交互增强
  说明：
  支持编辑区右键菜单；支持文字元素双击直接进入编辑态，减少在固定面板和工具栏之间往返。
- [ ] 组件管理面板重构
  说明：
  将组件列表改为编辑器右上角独立浮层，支持拖拽排序直接修改层级，并外化高频工具；同时优化组件列表、组件属性与时间轴之间的联动关系，并评估移除右侧“页面设置”模块。
- [ ] 缩略图自动截图同步
  说明：
  保存或切换到下一页时，自动截图当前 slide，并同步更新左侧缩略页封面。
