# Canvas Courseware VNext Plan

本文档用于记录下一版本的规划、范围边界和开发批次，不直接替代当前执行中的 `docs/TODO.md`。

## 版本定位

- 当前版本仍在收尾 `VNext-22` / `VNext-23` 的 review 与 `VNext-24` 页面设置逻辑优化。
- 下一版本聚焦“页面级配置闭环 + 文本高频编辑效率”。
- 下一版本目标是在不新增元素类型和新媒体类型的前提下，提升页面级配置的可理解性、回显一致性与文本编辑的就地操作效率。
- 下一版本继续遵守 UI 层与事件层解耦原则，不引入新的引擎耦合型实现。

## 核心功能范围

- [ ] VNext-25 页面设置入口统一与作用域提示
  说明：
  基于当前页面设置入口分散、页面级 / 组件级语义容易混淆的问题，统一页面设置入口形态、打开路径和默认反馈，并在抽屉内补齐当前页名称、页序与“仅作用于当前页”的作用域提示。
  需求拆分：
  1. 统一页面设置入口 icon、打开方式和默认反馈，避免不同位置触达体验不一致。
  2. 补齐当前页名称、页序与作用范围提示，降低页面级 / 组件级混淆。
  3. 保证从常规入口和快捷链路进入页面设置时的上下文一致。
- [ ] VNext-26 页面基础字段分组重排
  说明：
  将页面名称、页面尺寸等高频字段收敛为页面基础信息组，提升首屏可读性和查找效率。
  需求拆分：
  1. 将页面名称、尺寸等高频字段聚合为基础信息组。
  2. 重排字段顺序，优先展示高频配置。
  3. 控制默认展开层级，降低首屏认知负担。
- [ ] VNext-27 背景配置集中化
  说明：
  将背景色、背景图、填充方式和背景预览收敛为单一背景配置组，避免背景设置跨区域分散。
  需求拆分：
  1. 聚合背景色、背景图与填充方式。
  2. 增加背景结果预览或就近回显。
  3. 确保编辑态、缩略页与预览态的背景表达一致。
- [ ] VNext-28 快捷链路回跳与结果回显统一
  说明：
  围绕“设为背景”等页面级快捷操作，统一执行后的回跳规则、反馈方式与结果回显，避免不同入口得到不同状态表达。
  需求拆分：
  1. 明确快捷链路是否自动打开页面设置，以及打开到哪个分组。
  2. 统一操作成功后的状态反馈与回跳节奏。
  3. 保证页面设置抽屉、编辑画布、缩略页和预览态结果一致。
- [ ] VNext-29 页面设置信息减负
  说明：
  清理页面设置内外重复入口、重复文案和低频项平铺问题，进一步压缩页面设置首屏噪音。
  需求拆分：
  1. 移除低价值重复入口与重复说明。
  2. 将低频项折叠、下沉或延后展示。
  3. 保持高频操作首屏可达。
- [ ] VNext-30 `TextTool` 浮动文本工具条
  说明：
  在 `Text` 元素双击进入编辑态后，于元素上方显示就地 `TextTool`，承接颜色、字号、加粗、斜体等高频文本样式操作，缩短文字排版路径。
  需求拆分：
  1. `TextTool` 默认隐藏，仅在单个 `Text` 元素双击进入编辑态后显示。
  2. 工具条默认显示在文本元素上方，允许超出编辑区，不强制翻转到下方。
  3. 首批支持颜色、字号、加粗、斜体，且与右侧属性面板、自动保存和刷新恢复一致。
  4. 多选文本或非文本选中态不显示 `TextTool`。

## 暂不纳入下一版本

- 富文本扩展能力（段落、对齐、字间距、行高等完整文本面板）
- 表格、图表、音视频
- 组合 / 打散
- 协同编辑
- 模板市场
- 图片 / PDF 导出
- 多引擎同时接入
- 新元素类型与新媒体类型
