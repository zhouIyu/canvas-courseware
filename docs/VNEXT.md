# Canvas Courseware VNext Plan

本文档用于记录下一版本的规划、范围边界和开发批次，不直接替代当前执行中的 `docs/TODO.md`。

## 版本定位

- 上一版本已于 2026-04-10 结束并归档，详见 `docs/RELEASES.md` 中的 `Release 05`。
- 下一版本聚焦“开发态轻量诊断日志能力”。
- 下一版本目标是在不引入远程采集与重型平台依赖的前提下，为编辑器关键链路补齐统一、结构化、可回溯的最近日志能力，降低开发排障和测试复盘成本。
- 下一版本继续遵守 UI 层与事件层解耦原则，不引入新的引擎耦合型实现。
- 下一版本已完成当前开发与自测，当前以 `VNext-31` 作为等待用户 review 的唯一功能项。

## 核心功能范围

- [x] VNext-31 轻量诊断日志能力
  说明：
  围绕编辑器关键链路补齐统一的轻量诊断日志模型，优先服务开发排障和测试复盘；首批能力以本地内存保留、开发态查看与控制台输出为主，不引入远程采集。
  需求拆分：
  1. 新增统一 logger，支持 `debug / info / warn / error`，并在内存中保留最近若干条结构化日志。
  2. 首批接入命令执行、自动保存、导入导出、图片导入 / 替换 / 设为背景、页面背景写入与时间轴关键变更。
  3. 日志需携带最小上下文，例如 `projectId / slideId / nodeId / commandType / result`，避免记录整份文档或大体积图片数据。
  4. 开发态提供统一查看入口，保证开发和测试无需临时在多个文件中手工加 `console`。

## 暂不纳入下一版本

- 远程日志采集 / 上报
- 运营埋点、用户行为分析与审计日志
- 整份课件 JSON 或大体积图片数据的自动日志落盘
- 表格、图表、音视频
- 组合 / 打散
- 协同编辑
- 模板市场
- 图片 / PDF 导出
- 多引擎同时接入
- 新元素类型与新媒体类型
