# Canvas Courseware Requirements Pool

本文档用于记录产品侧持续收集中的需求池，仅做收集、整理和待评审记录，不等同于已排期开发任务。

## 待评审需求

- [ ] 右键菜单按上下文分离操作项：当已选中元素并打开对象级右键菜单时，不再显示“插入文本”等空白区插入类操作，避免对象操作与画布插入操作混在同一菜单中
- [ ] 时间轴引用一致性治理：当元素被删除后，关联的时间轴事件、触发对象和动画目标不应继续残留；需要重新梳理时间轴与元素之间的引用清理逻辑

## 需求细化记录

### RQ-CTX-02 选中元素时隐藏插入类右键菜单

- 背景：
  当前编辑区已经存在右键菜单能力。若元素已被选中，用户更关注对象级操作；这时继续显示“插入文本”等空白区插入类操作，会让菜单语义混杂，增加误操作风险。
- 目标：
  让右键菜单根据上下文展示更匹配的操作项。选中元素时，菜单聚焦对象操作；插入类操作只在合适的画布空白区场景出现。
- 用户场景：
  用户选中一个文本、图片或矩形后执行右键，希望看到复制、删除、层级或对象相关操作，而不是再看到“插入文本”等创建新元素的菜单项。
- 功能范围：
  当已选中元素并触发对象级右键菜单时，隐藏“插入文本”等插入类操作。
  当前默认理解为：该调整只作用于对象级菜单，不改变空白区域右键菜单中的插入能力；如需连带调整空白区菜单，可后续补充。
- 非功能约束：
  菜单项需保持“按上下文展示”的一致规则，避免同一组操作在不同状态下语义冲突。
  该需求属于右键菜单信息架构调整，不默认扩展为新增新的对象操作能力。
- 验收标准：
  选中元素后打开右键菜单，不再看到“插入文本”等插入类菜单项。
  对象级右键菜单只展示与当前选中对象相关的操作。
- 待确认项：
  插入类菜单项的具体范围是否包括“插入矩形”“插入图片”“插入图片框”等全部创建类操作。
  多选对象场景下是否沿用与单选对象一致的菜单策略。

### RQ-TL-01 删除元素后的时间轴引用清理

- 背景：
  当前元素删除后，时间轴中与该元素关联的事件没有同步删除，导致文档中出现失效引用。用户从画布上已经看不到该元素，但时间轴配置仍然保留旧目标，会让编辑态、预览态和后续维护都变得不可靠。
- 目标：
  建立“元素删除 <-> 时间轴引用清理”的一致规则，避免时间轴里残留指向不存在元素的触发器、动作或动画目标。
- 用户场景：
  用户删除一个已经参与时间轴编排的文本、图片或矩形后，希望时间轴同步进入正确状态，而不是继续保留无效事件。
- 功能范围：
  当元素被删除后，所有直接关联该元素的时间轴引用都需要同步处理。
  这里的关联至少包括：对象点击触发中的触发对象引用、步骤动作中的目标对象引用、动画中的 `targetId` 引用。
  需要重新梳理删除元素、删除图层、右键快捷删除、批量删除、以及图片转背景等会移除节点的入口，确保时间轴处理规则一致。
  当前默认理解为：删除元素后，应优先自动清理直接失效的时间轴引用；如需保留步骤壳体并提示用户二次修复，可后续补充确认。
- 非功能约束：
  时间轴与元素之间的联动必须遵守统一的数据一致性规则，不能在不同删除入口下出现不同行为。
  该需求优先解决引用一致性与编辑可靠性，不默认扩展为全量重构时间轴产品形态。
- 验收标准：
  删除元素后，时间轴中不再残留指向该元素的无效引用。
  编辑态时间轴面板、存储数据与预览态播放结果保持一致，不出现“画布已删除、时间轴仍引用”的状态错位。
  不同删除入口下，时间轴清理行为保持一致。
- 待确认项：
  当某个步骤因目标元素被删除而失去全部有效动作时，该步骤应自动删除，还是保留为空步骤并提示用户处理。
  当对象点击触发的触发对象被删除时，是自动删除该触发步骤，还是改为无触发对象的待修复状态。
  批量删除多个元素时，时间轴清理是否需要提供汇总提示。
